# ⚡️ uv Cheat Sheet

### 🚀 Running Code
Instead of `poetry run`, use `uv run`:
```bash
uv run sl --host localhost --port 9200 --auth 'admin:myStrongPassword123!' --disable-verify-ssl --disable-warn security list-users
```

---

### 📂 Virtual Environments
**uv** creates a `.venv` directory in your project root by default.

*   **Location:** `./.venv`
*   **Automation:** `uv` automatically creates/updates the environment on `uv run` or `uv sync`.
*   **No activation needed:** `uv run` handles the environment path for you.

---

### 📝 Core Commands

#### 1. Project Setup
*   `uv init` — Initialize a new project (replaces `poetry init`).
*   `uv sync` — Install dependencies and sync `.venv` (replaces `poetry install`).

#### 2. Managing Dependencies
*   `uv add <package>` — Add a dependency.
*   `uv add --dev <package>` — Add a dev dependency.
*   `uv remove <package>` — Remove a package.
*   `uv lock --upgrade` — Update all dependencies (replaces `poetry update`).

#### 3. Python Management
*   `uv python install 3.12` — Download and install a specific Python version.
*   `uv python list` — Show available Python versions.
*   `uv tree` — Display visual dependency tree.

---

#### ⚡️ Global Tools (pipx replacement)
Install and run CLI tools without polluting your project:
*   `uv tool install ruff` — Install a tool globally.
*   `uvx <package>` — Run a tool in a temporary, isolated environment (e.g., `uvx cowsay "Hello"`).
